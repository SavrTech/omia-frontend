Import and Export Guide

Import and Export OMIA Studio Projects Between Machines or Versions

May 1, 2026 • 4 min read

Fast Track

Export a Project ZIP from preview. Import that ZIP on the destination machine or app version. Your functional design moves over ready for the next round of editing.

If you need to continue a functional design on another machine or inside a different version of OMIA Studio, you do not need to rebuild it by hand. OMIA Studio lets you export the current project state as a Project ZIP and import that same package wherever you want to keep working.

The transfer flow is the same whether you are switching computers, handing a project to another workstation, or moving into a newer app release. Export from preview, import from the dashboard, and pick up from the latest saved project state.

Start from the project preview

Open OMIA Studio on the source machine or in the app version that currently holds the project you want to move, then open that project.

From the project preview, open the dropdown menu and choose Project ZIP. That tells OMIA Studio to prepare an export package for the full working project state.

Once the ZIP is ready, download or save it locally. This is the file you will move to the other machine or import into the other app version.

Import the ZIP on the destination setup

Open OMIA Studio on the destination machine or the app version you want to continue in. From the desktop dashboard, click the large plus button to reveal the import options.

Choose Import ZIP, select the exported Project ZIP, and let OMIA Studio bring the project into the local workspace.

Moving between machines

If you are transferring between two computers, copy the exported ZIP to the destination machine first. AirDrop, cloud storage, external drives, or any secure internal transfer method will work.

After the file is on the destination machine, import it from the OMIA Studio dashboard and continue editing there.

Moving between app versions

If you are upgrading or switching app versions, use the same export and import flow. Export the Project ZIP from the version that currently opens the project, then import it into the newer version.

This keeps the handoff explicit and avoids relying on background sync between installs.

What happens after import

Once the import completes, your functional design and its project files should open inside OMIA Studio ready for continued work. You are not starting from scratch. You are reopening the latest exported project state.

  • Your latest functional design layout comes over into the destination workspace.
  • The project files move with the export so you can continue from the same working state.
  • The imported project is ready for preview, iteration, and further development in the new environment.

Important transfer note

Projects do not automatically appear on another machine or in another install just because you sign in with the same account. The transfer happens when you explicitly export a Project ZIP and import that ZIP where you want the project to live next.

Best practice before you transfer

Export after your latest meaningful changes so the ZIP reflects the exact state you want to continue from. If multiple teammates or machines are involved, treat the ZIP as the handoff artifact for that point in time.

One flow, multiple handoff cases

Whether you are moving to a new laptop, a shared workstation, or a newer OMIA Studio release, the workflow stays simple. Export a Project ZIP from preview, import it on the target setup, and continue building without losing momentum.
